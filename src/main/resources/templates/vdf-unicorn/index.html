Inmetro Unicorn Application

Inmetro's Randomness Beacon is a web service, part of the SICCCiber project's effort to generate reliable and public sources of random numbers

About this page

This is a prototype of a random number service using the Unicorn protocol.

It allows for any user to publish their own seeds within a time frame. At last, the received values are combined by using a pre-defined VDF. The objective of this prototype is to propose a secure combination of beacons' results by using a Verifiable Delay Function(VDF). It has been built based on the proposal of Lenstra and Wesoloski (2018).

There are 3 possible states

  • Closed: Waiting for next 15-minute time slot for contributions
  • Open: Open for external contributions
  • Running: Verifiable Delay Funcition is running

Comments:

  • The Inmetro precommitment value (independent part) is combined with all contributions received and then a VDF is started a few moments before the next Inmetro beacon number is published
  • After the running state, the results are signed, stored and published together with all parameters used to execute the algorithm
  • A route has been added to the API that allows you to check all generated numbers
  • VDF is not performed if there are no contributions

The service is available via a public REST API.

Last Available Pulse:

First Pulse:
/beacon/2.0/vdf/unicorn/first
Example:

Verify Pulse:
/beacon/2.0/pulse/vdf/verify?y=<Y>&x=<X>&iterations=<iterations>
Example:

Post a value:
URI: /beacon/2.0/vdf/unicorn
Method: POST
Content-Type: Content-Type: application/json
Body:

{ "seed": "myseedValue", "description": "my description", "uri": "my uri" }



If a request for a Pulse results in no Pulse found, a 404 response is returned.
If a request for a Pulse is not in RFC 3339 format, a 400 (BAD REQUEST) is returned.
